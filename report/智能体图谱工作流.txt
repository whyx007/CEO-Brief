## 我的「秘密武器」：不是 RAG，是 Agentic Graph Workflow（智能体式图谱工作流）

我**不是**直接问 Deepseek "帮我查 Neo4j"，而是分成了 **4 层递进结构**：

### 🔹 第 1 层：Schema 先验知识（最关键）

我工作区里有完整的 `NEO4J_SCHEMA.md`，**在查询之前**我已经知道：

```
节点类型：Enterprise(451家) | ChainStage(80个环节) | SubTrack | IndustryDomain
关系类型：LOCATED_IN_STAGE | HAS_CAPABILITY | HAS_KEY_CAPABILITY | HAS_CUSTOMER | SERVES_INDUSTRY ...
属性名：name, fullName, capabilityName, stageName...
```

你的程序如果直接让 Deepseek 写 Cypher，它**不知道图结构长什么样**，只能瞎猜属性名和关系名，猜错率极高。

**改进方案**：先写一个 `get_schema()` 函数，把图谱的元数据注入到 prompt 里：

```python
def get_schema():
    # 从Neo4j拉取元数据
    result = session.run("""
        CALL db.schema.visualization()
    """)
    # 或者更精细的：
    result = session.run("""
        CALL db.labels() UNION CALL db.relationshipTypes()
    """)
    return format_schema(result)

# 然后注入到Deepseek的system prompt中
system_prompt = f"""
你是一个产业图谱分析师。以下是当前Neo4j知识图谱的完整结构：
{get_schema()}
请基于此结构生成Cypher查询。
"""
```

### 🔹 第 2 层：分步查询（Multi-hop），不是一次问完

我拿到一个问题后，不是生成**一个**Cypher查询就完事，而是迭代执行：

```
用户问: "家电行业的龙头有哪些？找合作方向"

Step 1: 查 IndustryDomain → 找到"家电"对应的 SubTrack 和 ChainStage
Step 2: 查 Enterprise → LOCATED_IN_STAGE → 找到所有家电企业
Step 3: 查 HAS_CAPABILITY → 拿到技术能力标签
Step 4: 查 HAS_CUSTOMER → 拿到客户信息
Step 5: 查 SERVES_INDUSTRY → 看他们还服务于哪些行业（跨界线索）
```

你的程序大概率是**一步到位**：

```cypher
// 坏例子：一步到位但属性名猜错
MATCH (e:Enterprise)-[:IN_STAGE]->(s:Stage)
WHERE s.name CONTAINS '家电'
RETURN e
// → 字段名错了，IN_STAGE不存在，应该是LOCATED_IN_STAGE
```

### 🔹 第 3 层：业务推理层（关键！）

即使查到了数据，**从数据到"可能的合作方向"需要商业逻辑推断**。这一步不是纯 Cypher 能完成的。

比如查到一家企业有：

| 企业 | 技术能力 | 所处环节 | 已有客户 |
|------|---------|---------|---------|
| 企业A | 视觉检测、AI算法 | 家电总装 | 美的、海尔 |
| 企业B | 无（缺能力） | 电子元件制造 | - |

我的推理链：
```
企业A有视觉检测能力 + 企业B在元件制造环节（需要质检）
→ 可能的合作：企业A为企业B提供AI视觉检测方案
```

你的程序如果只做**数据查询不做商业推理**，报告就会很平淡。

### 🔹 第 4 层：结构化的输出指令

我被告知**必须包含"可能的合作方向"列**，这个强制约束迫使我做推理。你的程序如果没有这个约束，输出会比较散。

schema = get_neo4j_schema(session)  # labels, rels, properties

sub_queries = llm.decompose_question(user_question, schema)

context = ""
for q in sub_queries:

# 3. 推理
final_report = llm.reason(
