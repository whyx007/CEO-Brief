# CEO参阅 - 开发任务排期单

## 文档目标
将 CEO参阅模块拆成可执行的研发任务，按角色分工、先后顺序和联调节点推进，方便前后端、UI/UX 和 QA 开始工作。

适用角色：
- `product-manager`
- `ui-ux`
- `backend-platform`
- `frontend-web`
- `qa-testing`
- `project-management`

---

# 一、交付目标

本轮交付目标是完成 CEO参阅 MVP 第一版，覆盖：
- 产经信息
- 目标信息
- 今日代办
- 天气情况
- 设置页（目标设置 + Prompt 设置）

并满足：
- 首页可展示
- 设置可保存
- 支持手动重新生成
- 四块内容独立容错

---

# 三、建议排期（5个工作日版本）

> 目标：先做出 CEO参阅第一版可用稿，作为首个落地功能。

- 确认不做历史版本、审批流、复杂权限

- 输出设置页线框结构

- 建立接口路由骨架
- 建立数据表或模型骨架

## Frontend
- 建立首页页面骨架
- 建立设置页页面骨架

# 四、前置依赖

## 必须先确认的事项
1. 目标设置字段范围
2. Prompt 三项字段最终名称
3. 天气接口来源或 mock 方案
4. 新闻输入来源的初版范围

## 容易卡住的点
- Prompt 是否允许自由编辑到什么程度
- 新闻来源到底是 mock、人工录入还是外部抓取
- 天气默认城市逻辑

# 六、联调顺序建议

建议严格按这个顺序：
1. 首页只联 `GET /today`
2. 设置页联目标设置
3. 设置页联 Prompt 设置
4. 接 `reset` 接口
5. 最后接 `generate`

## 原因
- 先渲染，再交互
- 先设置，再生成
- 先稳定结构，再打通动作

---

# 七、最小可演示标准

满足以下条件即可认定第一版可演示：
- 首页能展示 4 个板块
- 设置页能保存目标设置
- 设置页能保存 Prompt 设置
- 支持手动重新生成
- 单模块失败不影响整页打开

---

# 八、下一轮建议（第一版完成后）

第一版完成后，再进入：
- 首页视觉优化
- 来源跳转
- 生成历史
- Prompt 变更留痕
- 更细的待办规则优化

---

# 九、项目经理提醒

如果要保证 CEO参阅尽快真正落地：
- 不要在第一版继续加模块
- 不要把“新闻源自动化”做成前置依赖
- 不要把 Prompt 管理做成复杂系统
- 先交可读、可用、可演示的版本
